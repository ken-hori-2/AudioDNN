# 音声分類モデル 精度向上ノウハウブック

## 1. データ前処理の工夫

### 音声長の統一
- **工夫**: 音声の長さを3秒に統一し、エネルギーが最も高い部分を中心に切り取る
- **課題**: 短い音声ファイル（1サンプル未満）の処理が困難
- **解決策**: 最小FFTサイズ（256）未満の音声にはパディングを追加

### メルスペクトログラムの計算
- **工夫**: FFTサイズとホップ長を音声長に合わせて動的に調整
- **課題**: 固定パラメータ（n_fft=2048）が短い音声に対して大きすぎる
- **解決策**: `n_fft = max(256, min(2048, len(audio)))`で動的に調整

### 特徴量の正規化
- **工夫**: 特徴量を正規化して学習を安定化
- **課題**: 異なる音声間でのスケールの違い
- **解決策**: `features = (features - np.mean(features)) / np.std(features)`で正規化

## 2. データ拡張の工夫

### マスキング手法
- **工夫**: 時間マスキングと周波数マスキングを実装
- **課題**: マスク幅が大きすぎると特徴が失われる
- **解決策**: マスク幅を時間/周波数ステップの1/4までに制限

### ノイズ追加
- **工夫**: ガウシアンノイズを追加してモデルの堅牢性を向上
- **課題**: ノイズ強度が強すぎると特徴が歪む
- **解決策**: ノイズ強度を0.01から0.05の範囲でランダムに設定

### 拡張方法の選択
- **工夫**: 問題のある拡張方法（時間伸縮、ピッチシフト）を削除
- **課題**: 一部の拡張方法が短い音声でエラーを引き起こす
- **解決策**: より安全な拡張方法（マスキング、ノイズ追加）に焦点を当てる

## 3. クラス不均衡への対応

### オーバーサンプリング
- **工夫**: 少数クラスのサンプルを複製してデータ拡張を適用
- **課題**: 単純な複製では過学習のリスクがある
- **解決策**: データ拡張を適用した複製を生成

### クラス重みの計算
- **工夫**: クラスごとのサンプル数の逆数の平方根を使用
- **課題**: 単純な逆数では重みの差が大きすぎる
- **解決策**: 平方根を取ることで重みの差を緩和

### クラスごとの処理
- **工夫**: クラスごとに処理して正確なサンプル複製を実現
- **課題**: 全サンプルを一括で処理すると少数クラスが適切に処理されない
- **解決策**: クラスごとにインデックスを取得して個別に処理

## 4. モデルアーキテクチャの工夫

### 畳み込み層の設計
- **工夫**: 3層の畳み込み層を使用して特徴を段階的に抽出
- **課題**: 層が多すぎると過学習のリスクがある
- **解決策**: バッチ正規化とドロップアウトを追加

### 適応的プーリング
- **工夫**: 適応的平均プーリングで異なるサイズの入力に対応
- **課題**: 固定サイズのプーリングでは情報が失われる
- **解決策**: 入力サイズに応じて自動調整する適応的プーリングを使用

### ドロップアウト
- **工夫**: ドロップアウト率を0.5に設定して過学習を防止
- **課題**: ドロップアウトが強すぎると学習が遅くなる
- **解決策**: 畳み込み層と全結合層の両方に適度なドロップアウトを適用

## 5. 学習プロセスの工夫

### 早期停止
- **工夫**: 25エポック間精度が改善しなければ学習を停止
- **課題**: 早すぎる停止で最適モデルを見逃す可能性
- **解決策**: 十分な忍耐値（patience=25）を設定

### 学習率スケジューリング
- **工夫**: ReduceLROnPlateauで学習率を動的に調整
- **課題**: 学習率が小さすぎると学習が停滞する
- **解決策**: 精度が改善しない場合に学習率を半分に減少

### 最適モデルの保存
- **工夫**: 検証精度が最も高いモデルを保存
- **課題**: 最終モデルが最適とは限らない
- **解決策**: 各エポックで検証精度を評価し、最高精度のモデルを保存
